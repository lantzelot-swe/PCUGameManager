PCU (Project Carousel Usb) Game manager is an application for managing games and games information used by the carousel for TheC64, The Vic20 and TheC64 Mini. The source code is available on Github.

If you find any bugs please report an issue for it here: Issues

For ideas and discussions you can post here: Discussions

You will also find the latest release here (it will only have the PcuGameManager.exe file, you will be able to replace it in an existing folder).

 

Files and folder structure

 

PCUGameManager holds game information in an SQLite database, the pcusb.db file. Covers, screenshots and game files are in the covers, screens and games folders. The jre folder contains a JRE (Java Runtime Environment) which is needed to run the application. The Vice folder contains the Vice emulator (C64 and Vic20) and is used when running games.

A backup folder is also created when a backup of the database is created. Each backup gets a separate folder named by the date and time it was created.

pcusb.log contains debug logging and is useful to include when reporting bugs.

 

Database management

 

The pcusb.db file contains all information about the games. It also holds the different game views. The pcusb.db file can be deleted, the game manager will create an empty database if the file is missing.

You can create a backup of the database and all covers, screens and game files whenever you want with the backup db menu option. The backups are saved in the backup folder. A backup can also be restored from there easily with the Restore backup menu option. You can also delete all games from the database with the Delete all games menu option. Before deleting a backup will be created. When deleting all games, the defined game views will be kept.   

 

Game Views

 

There are two game views available by default: All games and Favorites. These cannot be edited or removed.

It is possible to create custom game views by pressing the .. button to the left of the game view dropdown box. A game view has a name and two sets of filters for different game fields.

The first set includes filters that must match, the second set contains filters where at least one must match. This corresponds to AND and OR conditions for the db query that is generated for the view.

Example: To create a game view that lists all C64 games where title starts with a or b you create a filter System contains text 64 in the must match set and two filters title begins with text a and title begins with text b in the any filter set.

 

Scraper

 

Use the Scrape.. button in the bottom left in the game details panel. You can scrape information for an existing game (useful if you want to update part of the information, e.g. screenshots) or when adding a new game.

The scraper needs to know from which web page to fetch information, so you need to provide the URL for the game you want to fetch information for. MobyGames.com is very well-structured so fetching the information for specific fields is easy. Some games have many screenshots, the scraper will only download the first 6 screenshots to select from.

The genre information on the site is a bit different than the ones supported by the game manager. A mapping is done if possible.

The description information is usually very lengthy, so after scraping you need to edit it. Its also possible that the screenshots are in a different resolution than the optimal one (320x200). Press the yellow warning icon under the screenshots to automatically crop them if you like.
